# 목적

1. 일일 회고를 작성하기 위해 사용한다.

# 언제 사용하는 가

1. 일일 회고 작성하기 위해서 사용한다.

# 일일 회고 작성 규칙

1. 먼저 ./dailyReview-template.md를 읽는다.
2. dailyReview-template.md의 형식에 맞춰 쓰는데, 첫 번째 파일 소개 부분은 생략하고 진행한다.
3. 파일의 이름은 {mm}-{dd}.md 여야하며, 한국 시간 기준으로 월요일~토요일까지가 1주일이다. 
4. 헤더 크기를 조절하지 않는다.
5. 해당 헤딩에 작성할만한 내용이 없다면, 그 섹션은 생략한다.
6. 자연스럽게 작성하며, 부자연스러운 강조 및 기호를 남발하지 않는다.
7. 사실이 확인되지 않은 내용은 추측하여 작성하지 않고, 정보가 부족하면 사용자에게 물어본다.
