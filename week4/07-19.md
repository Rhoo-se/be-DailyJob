
### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습:
  - MCP의 개념, mcp라는 말이 자주 등장 하는데 정확한 의미와 개념을 꺠달았다. 하나의 프로토콜로서 ai 모델을 서비스 내부에서 사용할 수 있도록 도와주는 프로토콜.
  - Figma MCP를 활용해 메인 페이지, 로그인, 회원가입, 경매 등록 페이지의 와이어프레임을 상세 사항을 입력해 직접 구성함.
  - 18일부터 작성하던 기능명세서를 완성함(인증/계정, 경매 등록·탐색·상세/입찰, 결제, 정산, 알림, 마이페이지, 거래 관리, 관리자 대시보드 전 영역).
- (오늘의 목표 중에서) 아직 끝내지 못한 부분:
  - 나머지 화면(경매 상세/입찰, 마이페이지, 관리자 등)의 와이어프레임은 미완성.
  - 기능명세서는 "전체 나열" 단계까지 완료. 각 기능의 우선순위/범위(MVP vs 이후) 구분은 아직 안 됨.

### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단:
  Figma를 손으로 직접 그리는 대신 Figma MCP를 통해 AI로 와이어프레임을 구성하기로 함.
- 그렇게 판단한 이유:
  - 초기 화면은 정교한 디자인보다 "구조와 흐름 확인"이 목적이라, 빠르게 여러 화면을 뽑아 검토하는 것이 더 중요했음.
  - 기능명세서의 항목들을 화면으로 옮기며 누락/모순을 조기에 발견하기에 유리.
- 고려했지만 선택하지 않은 대안:
  - Figma에서 수작업으로 와이어프레임 제작 → 속도가 느리고 초기 탐색 단계에 과함.
  - 와이어프레임 생략하고 바로 기능명세서만으로 진행 → 화면 단위 검증이 빠져 기획 구멍을 놓칠 위험.
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향:
  - 화면을 먼저 시각화하면서 기능명세서의 항목이 실제 화면에 어떻게 배치되는지 검증 → 이후 개발 시 화면-기능 매핑이 명확해짐.

### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상:
  Figma MCP를 처음 사용해, 원하는 화면을 얻기 위해 어떻게 지시(프롬프트)해야 하는지 감을 잡기 어려웠음.
- 처음 추측한 원인:
  도구 사용법 자체를 몰라서라고 생각함.
- 원인을 확인하기 위해 시도한 방법:
  Claude.ai / Claude Code 사용법과 MCP 개념을 먼저 학습한 뒤, 화면별로 상세 사항을 입력하며 결과를 반복 확인.
- 실제 원인:
  도구 사용법보다는 "무엇을 만들지(화면에 담을 요소)"를 구체적으로 전달하지 못한 것이 원인. 지시가 모호하면 결과도 모호했음.
- 해결 방법:
  각 페이지에 들어갈 요소를 상세히 명시해 입력 → 의도한 화면에 가깝게 구성됨.
- 다음에 비슷한 문제가 생기면 먼저 확인할 것:
  "도구를 모르는 문제"인지 "요구사항이 모호한 문제"인지 구분하기. AI 도구는 대개 후자인 경우가 많으므로, 입력(요구사항)을 먼저 구체화.

### 4. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지:
  기능명세서를 완성한 뒤, 그 항목들을 곧바로 와이어프레임으로 옮겨 화면 단위로 검증함.
- 그 접근이 도움이 된 이유:
  - 기능 목록(텍스트)만으로는 안 보이던 "화면에 어떻게 배치될지 / 어떤 게 빠졌는지"가 시각화하니 드러남.
  - 새 도구(MCP)를 학습만 하고 끝내지 않고 실제 산출물(화면)로 바로 연결함.
- 다음에도 반복하고 싶은 점:
  - 기능 정의 → 화면 검증의 흐름을 유지하기.
  - 새 도구는 개념 학습 후 곧바로 실제 작업에 적용해 체화하기.

### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)
- 비효율적이었던 접근/놓치거나 부족했던 점:
  기능명세서를 "전체 나열"까지만 완성하고, 각 기능의 우선순위(MVP/이후)나 구현 범위 구분을 함께 하지 않음. 결과적으로 항목 수가 매우 많아짐.
- 그렇게 된 원인:
  "필요할 것 같은 기능"을 폭넓게 나열하는 데 집중했고, "이번 범위에 실제로 넣을 것"을 거르는 관점이 빠졌음.
- 다음에는 다르게 해볼 구체적인 행동:
  - 기능명세서 작성 시 항목마다 우선순위/범위 태그(MVP·이후·제외)를 함께 표기하기.
  - 화면도 우선순위 높은 것부터 그려, 전체를 얕게 그리기보다 핵심 흐름을 먼저 완성하기.
